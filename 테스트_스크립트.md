# 문의하기 기능 통합 테스트 가이드

## 📌 목표
메인 홈페이지의 '문의하기' 폼 제출 → 관리자 페이지의 문의 목록에 데이터가 정상 표시되는지 확인

---

## 🔧 사전 준비

### 1. 백엔드 서버 재시작
```bash
# 백엔드 디렉토리 이동
cd /Users/nam/Desktop/웹/코드/backend

# package.json 확인 (스크립트 확인)
cat package.json | grep -A5 '"scripts"'

# PM2 사용 중인 경우
pm2 restart ecosystem.config.js

# 또는 직접 실행
node app/index.js
```

### 2. DB 연결 확인
```bash
# 백엔드 로그에서 다음 메시지 확인
# "[개발 모드] 서버가 작동되었습니다 : port 6006"
```

## 🧪 테스트 절차

### Test Case 1: API 직접 호출 (curl)
```bash
curl -X POST 'http://localhost:6006/v1/quickcontact/post' \
  -H 'Content-Type: application/json' \
  -d '{
    "qc_name": "테스트사용자",
    "qc_phone": "010-1234-5678",
    "qc_field": "고등학생",
    "qc_location": "빛솔본점",
    "qc_email": "test@example.com"
  }'
```

**기대 응답**: 성공 신호 (status 200 또는 유사)

---

### Test Case 2: 관리자 API로 데이터 확인
```bash
curl 'http://localhost:6006/v1/super/inquiry/all' \
  -H 'Authorization: Bearer [YOUR_ADMIN_TOKEN]'
```

**기대 결과**: 위에서 입력한 문의가 목록에 포함되어야 함
- `cst_title`: "테스트사용자"
- `cst_content`: 지점, 학년, 연락처, 이메일 정보가 포함됨
- `cst_phone`: "010-1234-5678"
- `cst_email`: "test@example.com"
- `cst_regtime`: 현재 시간 (실제 문의 시간)

### Test Case 3: UI를 통한 테스트 (브라우저)

#### 3.1 메인 홈페이지 접근
```
http://localhost:[frontend_port]
```

#### 3.2 '문의하기' 섹션 찾기
- 'Contact' 섹션으로 이동
- ThirdContactSection 폼 확인

#### 3.3 테스트 데이터 입력
| 필드 | 입력값 |
|------|--------|
| 지점 | 빛솔본점 |
| 이름 | 김테스트 |
| 연락처 | 010-9999-8888 |
| 학년 | 중학생 |
| 이메일 | kim@example.com |

#### 3.4 제출 및 확인
- "문의하기" 버튼 클릭
- 성공 메시지 ("성공적으로 문의 남겼습니다.") 확인

---

### Test Case 4: 관리자 페이지 확인
```
http://localhost:[admin_port]/admin
```

#### 4.1 관리자 로그인
- 관리자 계정으로 로그인

#### 4.2 문의 목록 탭 접근
- "사이트" → "문의" 또는 "문의내역" 탭

#### 4.3 데이터 표시 확인
다음 정보가 모두 표시되어야 함:

| 컬럼 | 예상값 | 설명 |
|------|--------|------|
| 답변여부 | 답변대기 | 새 문의이므로 |
| 문의분류 | 기본 | 기본 카테고리로 설정 |
| 문의제목 | 김테스트 | qc_name으로 설정됨 |
| 문의내용 | [상세정보] | 지점/학년/연락처/이메일/시간 포함 |
| 문의자 | 김테스트 | mem_nickname 없으므로 cst_title 사용 |
| 연락처 | 010-9999-8888 | qc_phone 매핑 |
| E-mail | kim@example.com | qc_email 매핑 |
| 작성일자 | [현재날짜] | 실제 현재 시간 기반 |

#### 4.4 상세 보기 클릭
- "답변내역 보기" 버튼 클릭
- 문의내용에 모든 정보가 포함되어 있는지 확인

## ✅ 성공 기준

- [ ] API 호출이 오류 없이 완료됨
- [ ] 관리자 목록에 새 문의가 보임
- [ ] 문의제목이 사용자명으로 표시됨
- [ ] 문의내용에 모든 정보(지점, 학년, 연락처, 이메일)가 포함됨
- [ ] 작성일자가 현재 시간으로 저장됨
- [ ] 연락처와 이메일이 올바르게 매핑됨

## 🐛 문제 해결

### 문제: "Failed to insert into wb_inquiry" 에러
**원인**: 데이터베이스 연결 오류 또는 테이블 구조 문제
**해결**:
```bash
# 백엔드 로그 확인
tail -50 backend.log

# DB 연결 설정 재확인
cat /Users/nam/Desktop/웹/코드/backend/app/config/config.development.js
```

### 문제: 관리자 목록에 문의가 보이지 않음
**원인**: 데이터가 `wb_inquiry`가 아닌 `bs_quickcontact`에만 저장됨
**해결**:
```bash
# DB에 직접 확인
# SELECT * FROM wb_inquiry WHERE cst_title LIKE '%테스트%';

# quickcontact.model.js 로그 확인
# console.error 메시지 확인
```

### 문제: 이메일이 관리자 목록에 보이지 않음
**원인**: 폼에서 `qc_email` 필드를 전송하지 않음
**해결**:
- 프론트엔드 콘솔(F12)에서 form submission 확인
- 네트워크 탭에서 요청 payload 확인

## 📝 추가 메모

- 빠른 문의는 `bs_quickcontact`와 `wb_inquiry` 두 테이블에 모두 저장됨
- 관리자 페이지는 `wb_inquiry` 테이블만 조회함
- 비회원 문의자의 경우 `mem_idx=0`으로 저장되며, `mem_nickname`이 없으므로 `cst_title`을 사용하여 표시됨
